SECTION VIII-F — TEXT REPLACE 純JS化

Ctrl+F で、浮かび上がる。

Phase 8-E で見送った 3 機能 (Ctrl+F バインド / フローティングパネル / ハイライト) を、 Gradio Script を廃して javascript/text_replace.js 1 ファイルの VSCode 風ウィジェットに置き換える。

Ⅰ.

なぜ作り直すのか

Phase 8-E の scripts/text_replace.py は Gradio Accordion パネルとして実装されたが、当初構想の 3 機能が未達のまま:

1

Ctrl+F バインド

現状: 手動で Accordion 開閉
プロンプト欄フォーカス中の Ctrl+F でパネルを開く。グローバル keydown フック。
2

フローティングパネル

現状: 通常 Accordion 内配置
画面に position:fixed で浮かぶダークテーマ対応パネル。
3

マッチハイライト

現状: テキストで前後文脈表示のみ
setSelectionRange で現在マッチを選択ハイライト + スクロール追従。

いずれもクライアントサイド JS でしか自然に実現できない機能。加えて現状の Gradio 実装は Python 往復 + .then(_js=) の Gradio 4.40 互換が未検証という弱点を抱える。よって純 JS ウィジェットに一本化する。

Ⅲ.

調査で判明したForge の仕組み

  • javascript/*.js は起動時に自動 <script> 読込 (modules/ui_gradio_extensions.py:javascript_html())。リポジトリルートの javascript/ が対象。
  • グローバル keydown の参考: javascript/edit-attention.js (Ctrl+方向キー)。フォーカス判定 target.matches("*:is([id*='_toprow'] [id*='_prompt'], .prompt) textarea")。
  • textarea 値を書き換えたら updateInput(target) (javascript/ui.js) を呼ぶと Gradio 状態が同期。
  • メインプロンプト DOM: #txt2img_prompt textarea / #img2img_prompt textarea。
  • CSS は JS から <style> 注入が最も自己完結的 (style.css 自動ロードは名前固定のため新規 CSS に使えない)。

Ⅵ.

text_replace.js の構成

  1. CSS 注入: ロード時に <style> を head に追加。フローティングパネル (fixed / 半透明ダーク背景 / 角丸 / シャドウ)。デフォルト非表示。
  2. DOM 構築: パネル (検索 input / 置換 input / 件数 N/M / ◀▶ / Aa 大小区別 / .* 正規表現 / 1件置換 / 全置換 / × 閉じる) を 1 度だけ生成し document.body 直下に append。
  3. 状態: activeTextarea / matches / currentIndex。
  4. 検索ロジック (純 JS、text_tools.py をミラー): findMatches / replaceOne / replaceAll / navigate(dir) (循環)。正規表現無効は try/catch で空配列。
  5. Ctrl+F ハンドラ: prompt textarea フォーカス時のみ preventDefault + パネル表示 + 検索 input にフォーカス。それ以外はブラウザ標準を素通し。
  6. ハイライト: currentIndex のマッチを setSelectionRange + スクロール調整 + 件数表示。
  7. Esc: 閉じて textarea にフォーカス復帰。
  8. 置換後: value 書換 → updateInput で Gradio 同期 → 再検索してハイライト更新。

リスクと緩和

Ctrl+F がブラウザ標準検索と衝突

prompt textarea フォーカス時のみ preventDefault。それ以外は素通し。

F-09 で確認
非フォーカス textarea の選択ハイライトが見えにくい

多くのブラウザは非フォーカス選択を淡色描画する。

navigate 時に一瞬 textarea へ focus→選択→search input へ復帰する方式を F1 で検証
updateInput 未定義環境

関数が無いと Gradio 同期が失敗する可能性。

typeof updateInput === 'function' ガード後に呼び、無ければ input イベント手動 dispatch にフォールバック
Gradio 動的再描画でパネル DOM が消える

Gradio コンテナ内に置くと再描画で消失。

パネルは document.body 直下に append しコンテナ外に隔離
JS ロジックに自動テストがない

純 JS のため L1 で覆えない。

text_tools.py + tests を挙動 spec として残置、JS はそれをミラー。重要ケース (正規表現無効/循環) を F-04/F-07 で手動確認
