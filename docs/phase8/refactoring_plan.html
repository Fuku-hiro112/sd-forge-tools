注意点と回避策

FIG-004 · CAUTION
dynamic-prompts との衝突

$varname= 構文と pyparsing が衝突。prompt_expander の UI で「dynamic-prompts は OFF 推奨」を明示。起動時に両方検出したら警告ログ。

p.all_prompts 書き換えと ADetailer

AlwaysOn 拡張同士の処理順序に注意。dynamic-prompts と同パターンを採用し、ADetailer / Hires.fix / forge-couple との併用 L2 シナリオを毎 Phase で確認。

Forge React で <script> が動かない

Text Replace の JS 注入は Blocks.load(_js=…) 経由で。gr.HTML 内の script タグは innerHTML 経由で実行されないため避ける。

ロールバック地点の管理

各 Phase 完了時に git tag（phase-8-a-done など）を打つ。事故時は git reset --hard <tag> で即時復帰。

AlwaysOn 拡張同士の連携

prompt_expander / vram_recycler / text_replace が 同一の p を共有。各拡張は自分の責務のみを変更し、他拡張の状態に依存しない設計を徹底。

v3b 削除のタイミング

Phase 8-D 完了直後の v3b 完全削除は危険。最低 1 週間 deprecation スタブとして残置し、ユーザー設定移行を確認してから削除。

VL-07 · 変更禁止項目 (Phase 8 で勝手に変えてはならない)
  • vars/variables.json の schema (schema_version: 2, categories[].children[].variables[] 入れ子)
  • 変数エントリのフィールド名 (name / positive / negative / updated_at / id)
  • name の $ プレフィックス含む保存形式 (Manager 慣習、Forge 側で strip)
  • 値末尾 ; の strip ルール (Hotfix-2)
  • updated_at の ISO8601 UTC 形式 (秒精度 + Z 末尾)
  • API URL パス (/vram_safe_batch/api/{set_jobs|status|peek_jobs|sync_variables|variables|variables/events})
  • SSE ペイロード形式 ({type: "external_change", mtime: float})
  • file watcher の polling 間隔 (1.0 秒)
改善提案がある場合の手順:
  1. 影響範囲を分析 (Manager / Forge / Migration 全部)
  2. 後方互換性の保証方法を提示
  3. ユーザーに確認して承認を取る
  4. その後にのみ実装

実装エージェントが独断で変更することを禁ずる。
