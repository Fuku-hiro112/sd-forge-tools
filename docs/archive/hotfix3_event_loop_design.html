🔴 Hotfix-3: WebUI 起動時 event loop エラー

原因 / 修正方針 / 影響範囲 を 1 ページにまとめた設計確認資料

!1. 症状

WebUI 起動時、_register_api コールバックが以下のエラーで失敗。 API（/sync_variables など Phase A/B/C で追加した分）は全て未登録になり、Variable Manager 同期が動かない。

*** Error executing callback app_started_callback for ...vram_safe_batch_v3b.py
    File "...vram_safe_batch_v3b.py", line 259, in _register_api
        asyncio.create_task(_watch_variables_file())
    RuntimeError: no running event loop

!2. 原因

_register_api(demo, app) は WebUI/Forge から 同期コールバックとして呼ばれる（event loop の外側）。 asyncio.create_task() は実行中の event loop が必須なので、ここで直接呼ぶと RuntimeError。

Phase B (commit 298d23c0) で file watcher を入れた時、event loop の起動タイミングを考慮せずに _register_api 末尾で asyncio.create_task を直接呼んでしまったのが直接原因。

✓3. 修正方針

FastAPI の startup イベントハンドラを介して、event loop が起動した直後に watcher タスクを生成する。

修正前（NG） 修正後（OK） 
asyncio.create_task(
    _watch_variables_file()
)
 
@app.on_event("startup")
async def _start_watcher():
    asyncio.create_task(
        _watch_variables_file()
    )
動作タイミング:
  1. WebUI 起動 → _register_api 同期呼び出し
  2. app.on_event("startup") でハンドラ登録（タスクは作らない）
  3. uvicorn が FastAPI を起動 → event loop 内で startup ハンドラが呼ばれる
  4. このとき初めて asyncio.create_task が成功し、watcher タスク開始
注意: app.on_event("startup") は FastAPI 0.96+ で deprecation 警告が出る（最終的に lifespan へ移行が推奨）。 Forge の FastAPI は古めのため当面これで動かす。将来 Forge が FastAPI を上げたら lifespan パターンに移行する余地あり。

注意事項

  • watcher タスクの起動が uvicorn の startup イベント発火後に遅れる（数百 ms 程度、実用上は無視可）
  • on_event("startup") deprecation 警告が将来のログに出る可能性

変化なし

  • 変数展開ロジック・履歴・シードモード等の機能挙動
  • API のエンドポイント仕様（同じ URL/レスポンス）
  • テスト数（127 件）

i6. L2 検証手順

  1. webui-user.bat 起動
  2. 起動ログに RuntimeError: no running event loop が 出ないこと
  3. 起動ログに以下が表示されること: 
    [VRAM Safe Batch] API endpoints registered: /set_jobs /status /peek_jobs /sync_variables /variables /variables/events
  4. Variable Manager (sd_variable_manager.html) を開き、変数を 1 個編集 → 500ms 以内に vars/variables.json が更新される
  5. 外部エディタで vars/variables.json を直接編集して保存 → 1〜2 秒で Manager UI に反映、トースト「🔄 Forge 側変更を取り込みました」
